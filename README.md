# Продуктовая аналитика маркетплейса
## Описание проекта
В данном проекте проведен комплексный анализ данных e-commerce маркетплейса. Цель исследования — изучить поведение пользователей, рассчитать ключевые продуктовые метрики и оценить влияние различных бизнес-инициатив на рост заказов.

## Технологический стек
* **Python** 
* **Pandas** 
* **Seaborn & Matplotlib**

## Основные этапы работы

### 1. Исследование данных и расчет метрик
* Проведен разведочный анализ данных (EDA).
* Приведение типов данных к формату `datetime` для временного анализа.
* Объединение данных о заказах (`orders`) с данными о пользователях (`users`) для идентификации уникальных клиентов.
* Рассчитаны базовые метрики: динамика общего объема продаж, количество активных пользователей (MAU) и средний чек.

### 2. Когортный анализ (Retention Rate)
* Сформированы месячные когорты по дате первой покупки.
* Рассчитан Retention в оформление заказа. 
* Выявлена динамика удержания: зафиксирован низкий уровень повторных покупок, что указывает на специфику товара или необходимость внедрения CRM-маркетинга.

### 3. Оценка бизнес-гипотез (Impact Analysis)
С помощью фреймворка приоритизации (ICE) были проанализированы три направления роста (на данных с июня 2017 года):
1. **Техническая оптимизация:** Борьба с отменами заказов и ошибками процессинга.
2. **Логистика:** Анализ влияния задержек доставки на лояльность.
3. **Платежные системы:** Прогноз влияния новых способов оплаты на конверсию в повторный заказ.

## Ключевые выводы и рекомендации
* **Приоритет №1:** Устранение багов в процессинге заказов (показало наибольший потенциальный прирост заказов).
* **PMF и лояльность:** Высокий процент клиентов совершает только одну покупку. Рекомендуется исследовать целевую аудиторию и причины отказа от повторных заказов.
* **Масштабируемость:** Несмотря на низкий Retention, общий объем продаж и база активных пользователей растут, что говорит о хорошем притоке нового трафика.
